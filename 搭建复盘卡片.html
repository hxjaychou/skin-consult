02 · DATABASE

创建 Supabase 数据库

1

注册并登录

用 GitHub 账号登录 supabase.com，不要用邮箱注册

2

创建新项目

区域选新加坡或东京，不要选美国

3

执行建表脚本

在 SQL Editor 中粘贴 setup-supabase.sql 全部内容，点 Run

4

获取 API 信息

Settings → API，复制 Project URL 和 anon public Key

踩坑 #1：注册方式

用邮箱注册后，后面 Cloudflare 也要 GitHub 授权，来回切换很麻烦。建议一开始就用 GitHub 账号登录所有服务。

踩坑 #2：区域选择

选了美国区域，访问速度慢到怀疑人生。改成新加坡区域后速度提升 3 倍。客户在中国，一定选亚洲区域。

踩坑 #3：表结构不完整（大坑）

SQL 脚本只建了 5 个字段，但网站需要 13 个。提交表单一直报错，以为是权限问题，查了半小时才发现是字段不够。解决：DROP TABLE 重新跑完整脚本。

03 · CONFIGURATION

配置网站代码

1

打开 index.html

用文本编辑器（推荐 VS Code）打开

2

填入 Supabase 信息

找到 URL 和 KEY 两行，替换为你自己的值

3

修改管理密码

找到 DB = '8866'，改成你自己的密码

4

本地测试

双击 index.html 在浏览器中打开，测试全部功能

踩坑 #4：API 信息填反

把 URL 填到了 KEY 的位置，提交表单一直报 401。检查方法：搜索 supabase.co 确认 URL，搜索 eycJ 确认 Key。

踩坑 #5：数据库未初始化

本地测试提交表单没反应，控制台报错找不到 consultations 表。原因是 Supabase 项目还没初始化完成，SQL 脚本没执行。

04 · GITHUB

上传到 GitHub

1

创建新仓库

Repository name 填 skin-consult，选 Public，不要勾选 Add a README

2

上传文件

Add file → Upload files → 拖入 index.html

3

提交

确保勾选 Replace all files，然后 Commit changes

踩坑 #6：文件没替换成功（最崩溃）

没看到 Replace all files 选项，以为同名文件会自动替换。结果 GitHub 上还是旧文件，网站打开是旧版本。查了 API、试了 git push，折腾一小时才发现是上传时没正确替换。

解决方法

用 GitHub 编辑器：打开 /edit/main/index.html → 全选删除 → 粘贴新内容 → Commit。这种方式最可靠。

踩坑 #7：浏览器缓存

上传成功后网站还是旧版本？强制刷新：Ctrl+Shift+R（Windows）或 Cmd+Shift+R（Mac）。

05 · DEPLOYMENT

部署到 Cloudflare Pages

1

登录 Cloudflare

dash.cloudflare.com，可以用 GitHub 账号登录

2

创建 Pages 项目

Workers & Pages → Create application → Pages → Connect to Git

3

连接 GitHub 仓库

选 GitHub → 授权 → 选 skin-consult → Begin setup

4

配置部署

Build settings 全部留空，Build output directory 填 /

5

等待部署

1-2 分钟后状态变 Success，得到 xxx.pages.dev 域名

踩坑 #8：两步验证卡住

GitHub 授权时需要两步验证，手机没在身边卡了好久。教训：提前确保手机在身边。

踩坑 #9：构建命令填错

在 Build command 里填了 npm install，结果部署失败。纯 HTML 网站不需要构建命令，全部留空。

踩坑 #10：文件路径错误

部署完成后打开是空白页。原因是 index.html 不在仓库根目录。确保文件在根目录。

06 · LESSONS

六条经验

1

先小后大

先做最小可用版本，测试通过再加功能。我一开始想做全部功能，结果到处出错。

2

每步验证

建表后测插入，上传后本地测，部署后查基本功能。别等全做完才测试。

3

善用 F12

Console 看错误，Network 看请求，Elements 看结构。开发者工具是你最好的朋友。

4

缓存是万恶之源

问题不是代码错，是浏览器在骗你。养成强制刷新的习惯。

5

版本备份

每次修改前备份当前版本。我好几次改坏了又没备份，只能重头再来。

6

不要怕问

遇到问题先搜索，搜索不到就问 AI。技术只是工具，你的专业服务才是核心。
